  • General Motors

    General Motors is a multinational automotive manufacturing company. Having multiple IT centers across the USA, many of the teams are focused on building out tooling to provide and maintain data useful to the manufacturing process.

    Company Website More

    Experience

    During my time at General Motors, I worked in the data quality organization. My team was responsible for helping manage the data around manufacturing, meaning a lot of the information that came up around defects the manufacturing process, they would be recorded in FMEA files (Failure Modes Effects Analysis).


    The majority of my work was centered around migrating all the data in our existing FMEA host into a different one. Given the existing FMEA system did not have a way for our team to mass export data, the only option was to extract each FMEA file as CSV, similar to how an individual user would export data. The data then had to be interpreted and trasformed such that it could be written into the new system.


    The tech stack for this role was very interesting. We had a lot of freedom to choose what we wanted to use because there was no precedent set, and all we really needed was a black box solution. Since exporting the data was the first issue we had to solve, we had to write a bot using java and a selenium driver to take all the actions of the user would to export data. This data then had to be saved into a server file storage, then read into, transformed, and written into the new system, which the entry point into that system was via API. This component was written in python due to the existing libraries and ease of use.
